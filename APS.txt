INTRODUÇÃO

	O desenvolvimento desta APS (Atividade Prática Supervisionada), se dará neste primeiro momento tendo como base o modelo conceitual de uma aplicação que “será criada”. Para que entendamos mesmo que de forma superficial o tema desta atividade prática supervisiona precisamos antes esclarecermos como se dará o “funcionamento”, ou seja, do que se trata a aplicação.
	A aplicação em questão tem como objetivo principal e claro permitir que um usuário comum possa ter acesso a qualquer produto através de um aplicativo, ou seja, possa através de um simples aplicativo conseguir pedir, receber e realizar a compra de algum produto.
	A ideia central é que existam os fornecedores dos produtos comprados pelo usuário, que serão: padarias, mercearias, açougues, supermercados dentre outros. Tendo como objetivo permitir que o usuário possa buscar o produto que está mais perto do mesmo, e também aquele que é o mais barato. Desta forma o usuário terá livre escolha de qual produto ele irá consumir ou comprar. Sendo que obviamente fica mais fácil para os fornecedores de tais produtos fazer a entrega do mesmo quando o usuário se localiza mais perto do local onde ele solicitou a compra.
	Existem muitos fatores que ainda podem ser desenvolvidos dentro dessa aplicação e que podem assim trazer uma otimização da mesma. Por hora os temas que estão intimamente relacionados com a aplicação são: Integração de Sistemas, Computação Móvel, Sistema de Arquivos Distribuídos, Gestão de Sistema Distribuído, Web Services e IOT (Internet Of Things).
	Esta atividade prática supervisionada, tem como objetivo abordar a aplicação tendo em vista o Web Service como foco principal, ou seja, como os Web Services agiriam em tal sistema, ambiente e aplicação. Esta é a missão desta “dissertação”: ambientar os Web Services dentro desta aplicação que em tese será desenvolvida. 
Se analisarmos tudo isto de uma forma externalizada, ou seja, analisar a questão da logística, da economia, da localização veremos que esta aplicação faz um bem ao seu usuário que pode pedir aquilo que deseja mesmo sem sair de casa, e escolhendo qual produto melhor atender as suas expectativas.
	A aplicação será acessada pelo usuário através de um dispositivo móvel e computação móvel, por esse e outros motivos se justifica utilizar para essa aplicação o conceito de sistemas distribuídos:
	“Nossos exemplos são baseados em redes de computadores que nos são familiares e amplamente difundidas: a internet, intranets e a emergente tecnologia das redes baseadas em dispositivos móveis. Eles foram planejados de modo a exemplificar a ampla gama de serviços e aplicações suportadas pelas redes de computadores e para iniciar a discussão sobre os problemas técnicos que fundamentam a sua implementação.” Retirado de (COULOURIS,George)
	Através do trecho acima podemos perceber que a aplicação tem características que justificam fazê-la (desenvolvê-la) com sistemas distribuídos uma vez que estará acessando rede e será acessada por dispositivo móvel.
	Por ser uma aplicação feita com sistema distribuídos ela possui muitas formas de ser entendida, e como já explanado anteriormente os principais assuntos que estarão rondando a mesma são: Integração de Sistemas (Como os sistemas de fornecedor, usuário, entrega, estoque estão relacionados), Computação Móvel e Internet (Como este tópico está “embutido” na aplicação), Sistema de Arquivos Distribuídos (Responsável por organizar, armazenar, recuperar, nomear, compartilhar e proteger arquivos (Sistemas Distribuídos; Sistemas de Arquivos Distribuídos) ), Gestão e orquestração de Sistemas Distribuídos, IOT, e Serviço Web que é o tema que será mais futuramente abordado nesta atividade prática supervisionada.
	Tendo em vista que este trabalho tratará de como o Web Service ou Serviço Web agirá na aplicação temos:
“Web Services são aplicações autocontidas, que possuem interface baseadas em XML e que descrevem uma coleção de operações acessíveis através de rede, independentemente da tecnologia usada na implementação do serviço” (W3C, 2004).
	Quando se trata de serviços web, comumente falamos que eles servem como um integrador de sistemas que permitirá que os sistemas, apesar de estarem em linguagens e plataformas diferentes poderão se comunicar mesmo assim.
	Basicamente o que podemos destacar mesmo que de forma simplificada dos Serviços Web é que eles são tecnologias construídas para comunicação entre sistemas sendo que é projetado para suportar interoperabilidade entre as máquinas numa rede. (FURTADO, Rômulo)
	Foi aproximadamente no final da última década antes da virada do milênio que se tornou necessário a padronização da comunicação que aconteciam entre diferentes plataformas, sendo elas o Computador Pessoal, Mainframe, Computador Mac, Windowns e Linux e também das linguagens de programação como C#, Java, PHP entre outras linguagens. Para isso, alguns padrões foram propostos para resolver este problema, como por exemplo o DCOM e CORBA, porém esses padrões não obtiveram um grande êxito naquilo que era sua tarefa principal; a utilização do RPC (Remote Procedure Call) por exemplo quando utilizado para a comunicação em diferentes nós poderia apresentar alguns problemas de segurança, e isso impossibilitaria por exemplo o seu uso na Internet devido aos requisitos de mensagem. (MORAES, Juliano)
	Com as informações acima podemos observar e salientar qual será o papel do Web Service dentro da aplicação: Será responsável por fazer a integração entre os diferentes sistemas para que isso apareça para o usuário como algo único. Assim mesmo que um fornecedor tenha linguagem diferente ao de um entregador ou de outro fornecedor, através do web services eles poderão conversar, ou seja, ter algum tipo de comunicação, desta forma o sistema poderá estar integrado para o usuário
